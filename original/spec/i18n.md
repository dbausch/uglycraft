# Spec: i18n — translation support via FPC resourcestring + gettext

## Status

- [x] Drop `-Mtp`; add `{$H+}` directive; remove flag from build command
- [x] `resourcestring` section added; `YesKey`/`NoKey` converted to `var`; all German
  string literals replaced with resourcestring names (English defaults; no

  `ShowItemDescriptions` and `DrawBorder` updated

---

## Motivation

All user-visible text is hard-coded German. The game is published on itch.io.
English-speaking players cannot play it. Translation is added via FPC's
`resourcestring` mechanism (compile-time extraction of string literals into a
`.rsj` JSON table) combined with the FPC `gettext` unit
(`TranslateResourceStrings`, `GetLanguageIDs`) for runtime locale switching.

**Source language: English.** `resourcestring` defaults are English. German is
the first `.mo` translation. Users whose system locale is unrecognised see
English.

## Drop `-Mtp` (commit 1, separate concern)

The code was ported from Turbo Pascal but no longer relies on any TP-specific
behaviour:

- Typed constants are never reassigned at runtime (so no write-at-runtime issue).
- No TP-only syntax is used.

Add `{$H+}` at the very top of `UGLI_2.pp` (before `program`):

```pascal
{$H+}
program UGLI_2;
```

`{$H+}` makes `String` an `AnsiString` explicitly; without it the semantics
depend on the active mode silently. The existing byte-based UTF-8 handling in
`Draw` and `UTF8Cols` works identically with `AnsiString` (`Length` = byte count,
`S[I]` = byte access).

Remove `-Mtp` from the build command in `pyproject.toml`:

```toml
# before
cmd = "cd original && fpc -Mtp -Fuuos UGLI_2.pp"
# after
cmd = "cd original && fpc -Fuuos UGLI_2.pp"
```

Verify: `poe build-original` compiles; `poe run-original` plays through a full
level without regressions.

remove `Name: String[40]` from `TItemData` (retaining `Ch` and `Fg` only),
and introduce `GetItemName(I)` instead.

### `YesKey` / `NoKey`

Currently typed constants (`[Ord('J'), Ord('j')]` and `[Ord('N'), Ord('n')]`).
The English dialog prompt "Y / N" requires Y/y to be accepted. Convert both to
`var` declarations and populate them at startup after translation loading.

### Add `DrawItemName` (add before `DrawBorder`)

The bottom border row (row `FieldH = 20`) has:
- **Left**: `DrawLives` at col 3 — `'LIVES XX'` (8 display cols, ends at col 10)
- **Right**: `DrawBlocks` at col 68 — `'BLOCKS XXXX'` (11 display cols, cols 68–78)
- **Center safe zone**: cols 12–66 (55 display cols)

`DrawItemName` draws the current item's name centered within this safe zone,
padded with spaces to cover any previously longer name:

```pascal
procedure DrawItemName;
const
  Fg = CounterFg;
  Bg = CounterBg;
  ZoneStart = 12;
  ZoneEnd   = 66;
  ZoneW     = ZoneEnd - ZoneStart + 1;  { = 55 }
var
  Idx, NameW, Pad, I: Integer;
  Name, Padded: String;
begin
  Idx := ItemNo;
  if (ItemNo = 9) and (Level = 9) then Idx := 10;
  Name := GetItemName(Idx);
  NameW := UTF8Cols(Name);
  Pad := (ZoneW - NameW) div 2;
  Padded := '';
  for I := 1 to Pad do Padded := Padded + ' ';
  Padded := Padded + Name;
  while UTF8Cols(Padded) < ZoneW do Padded := Padded + ' ';
  Draw(ZoneStart, FieldH, Fg, Bg, Padded);
end;
```

Add `DrawItemName` call at the end of `DrawBorder` (after `DrawBlocks`).

`DrawItemName` also called after `ItemNo := ItemNo + 1` in the main loop
pickup handler (before `goto StartLevel`), so the border updates live:

```pascal
    ItemNo := ItemNo + 1;
    ItemX := 0;
    DrawItemName;
    if ItemNo = 10 then
      begin …
```

(`LevelComplete` and `PlayerCaught` both trigger `PrepareLevel` → `Redraw` →
`DrawBorder` → `DrawItemName`, so no explicit call needed there.)

## Locale detection and `YesKey`/`NoKey` init (commit 4)

Add `gettext` to the `uses` clause. At the start of `Init` (before `Randomize`):

```pascal
var Lang, FallbackLang: String;
begin
  GetLanguageIDs(Lang, FallbackLang);
  if Length(Lang) >= 2 then SetLength(Lang, 2);
  if FileExists('translations/' + Lang + '.mo') then
    TranslateResourceStrings('translations/' + Lang + '.mo');
  YesKey := [Ord(UpCase(sYesChar[1])), Ord(LowCase(sYesChar[1]))];
  NoKey  := [Ord(UpCase(sNoChar[1])),  Ord(LowCase(sNoChar[1]))];
  Randomize;
  { … rest of Init … }
end;
```

`GetLanguageIDs` returns strings like `'de_DE'`; `SetLength(Lang, 2)` truncates
to `'de'`. `FileExists` returns false if the `translations/` directory does not
exist → English defaults stay. The `YesKey`/`NoKey` init runs after translation
so German gets `[Ord('J'), Ord('j')]`, English gets `[Ord('Y'), Ord('y')]`.

`UpCase`/`LowCase` are in the `SysUtils` unit (or CRT for single chars). If
not available, use: `Ord(sYesChar[1]) or $20` for lowercase,
`Ord(sYesChar[1]) and $DF` for uppercase (works for ASCII A–Z).

```bash

poe build-original

# 2. Generate PO template (requires rstconv from FPC tools)

cp original/translations/UGLI_2.pot original/translations/de.po
```
