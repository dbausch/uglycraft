# UGLI 2 — Internationalisation Reference

## Resourcestring List

All 25+ user-visible strings are `resourcestring` (English defaults). FPC extracts them to `UGLI_2.rsj` at compile time.

| Group | Identifiers |
|---|---|
| Item names | `sItemName1`–`sItemName10` |
| Yes/No chars | `sYesChar` ('Y'), `sNoChar` ('N') |
| HUD labels | `sLabelLevel`, `sLabelScore`, `sLabelLives`, `sLabelPauses`, `sLabelBlocks` |
| Key help row | `sKeys1`, `sKeys2`, `sKeys3` |
| Help screen | `sHelpTitle`, `sHelpArrows`, `sHelpSpace`, `sHelpP`, `sHelpEsc`, `sHelpF1`–`sHelpF5`, `sHelpEnd`, `sHelpHome` |
| Story | `sStoryTitle`, `sStoryText` |
| Item descriptions | `sItemListTitle`, `sItemInstTitle`, `sItemInstText` |
| Dialogs | `sLevelPrefix`, `sGameOver`, `sYouWon`, `sRemoveBlocks`, `sPlayAgain`, `sYesNo` |
| High score | `sFirstName`, `sLastName`, `sScoreEntry`, `sHSFileError` |
| Intro | `sIntroBrand`, `sIntroWelcome` |
| Shared | `sPressKey` |

## YesKey / NoKey System

After translation is loaded, `Init` builds two `set of Byte` for matching keypresses:
```pascal
YesKey := [Ord(UpCase(sYesChar[1])), Ord(LowerCase(sYesChar)[1])];
NoKey  := [Ord(UpCase(sNoChar[1])),  Ord(LowerCase(sNoChar)[1])];
```

Both uppercase and lowercase versions are included. Dialog loops use `Byte(Code) in YesKey`. In German: `sYesChar = 'J'` → YesKey contains both `Ord('J')` and `Ord('j')`. The displayed `sYesNo` string must stay consistent with `sYesChar`/`sNoChar` or the prompt will show different letters than the game accepts.

## Translation Discovery

`Init` calls `GetLanguageIDs` (FPC `gettext` unit) to get the two-letter ISO locale code, then probes (later overrides earlier):
1. `<exedir>/translations/<lang>.mo`
2. Each `XDG_DATA_DIRS` component + `/locale/<lang>/LC_MESSAGES/UGLI_2.mo`
3. `XDG_DATA_HOME/locale/<lang>/LC_MESSAGES/UGLI_2.mo` (default `~/.local/share`)

If no `.mo` is found, English resourcestring defaults are used.

## Adding a Language

1. Build the game → generates `UGLI_2.rsj`
2. `poe make-pot` → refreshes `translations/UGLI_2.pot`
3. Copy `UGLI_2.pot` to `translations/<lang>.po`
4. Fill in `msgstr` values (ensure `sYesChar`/`sNoChar` and `sYesNo` are consistent)
5. `msgfmt translations/<lang>.po -o translations/<lang>.mo`
6. Place `<lang>.mo` in a `translations/` subdirectory next to the binary

`GetItemName(I)` is the only access point for translated item names — do not add a `Name` field to `TItemData`.
